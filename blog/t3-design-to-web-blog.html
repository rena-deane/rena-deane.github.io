Design to Web

Responsive Websites

Wikipedia states that responsive web design is an approach aimed at crafting sites to provide an optimal viewing and interaction experience across a wide range of devices (from desktop computer monitors to mobile phones). Basically put, a website that works on any size screen.

Responsive websites are important given the range of mobile devices these days and people opting to use them for browsing the web.

Mobile First Design

Mobile first design is where you initially design the site to fit a mobile device such as a smart phone. Then @media queries are added so that we can adjust content of a page based on the screen size.

Through practice it's easy to see why we do mobile first design as it gets more complicated to manipulate a page from a desktop design down to a mobile sized site. It also helps you make important decisions about structuring your content.

What are Frameworks? Pros and Cons

Frameworks in this context is a set structure of files and folders of standardised code that can be used as a foundation to start buildind a site.

Frameworks save alot of time as we don't have to start from scratch. It also takes care of cross-browser functionality so we don't have to worry about browser specific bugs. It gives you clean code so that you can easily identify elements and layouts. Most of off, frameworks are free to use.

However there are some restrictions with frameworks. You're restiction to that set of grids and code and it limits what you can design. If you really need to customise something then you can change it but it requires extra effort and time. During this process you will often discover that code that you don't need and it can take up alot of time.
