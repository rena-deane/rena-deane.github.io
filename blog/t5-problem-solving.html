Problem Solving

What problem?

I wanted to add a listener to my element. But the selector I was adding to my .js file wasn’t grabbing the correct element.

What problem solving techniques did I use?

I knew it wasn’t grabbing because I kept getting an error message saying it was undefined.

  • Googling error messages: My searches returned mostly jQuery solutions and/or the JavaScript solutions were too specific to a particular scenario it was hard to dissect and find the relevant code.
  • Try Something: I kept trying different things but my code was all over the place. I needed a place to track the results of my code.
  • Reading error messages and using the console to try new lines of code: This was the best solution for me - working in the console. There was a lot of trial and error but I got the hang of selecting the correct elements and rarely received errors. The console is really helpful and a problem solving technique I will use many times in the future.
How did I feel and what did I learn?

The process was extremely frustrating but extremely rewarding once I discovered the console and started getting the results I wanted.

Never give up because there is a technique that will help you get the answers you’re looking for. For me it is a mix of googling, reading error messages, asking for help from peers, coaches or friend and in other instances the rubber duck method.

Problem solving techniques and process. Am I confident?
  • Pseudocode - If pseudocoding is explaining what I want my code to do, sequentially, then I’ve started using this technique. Wouldn’t say I’m confident.
  • Rubber ducky method - When I was stuck on one particular problem I made a screen recording to send for my friend for advice. When I rewatched the video of me explaining what I wanted to do and why I thought it might not work I actually identified some errors myself. I think it’s a similar concept to the rubber ducky.
  • Reading error messages - Almost confident. Still discovering new errors and capabilities.
  • Console.logging - one of my favourite techniques.
  • Googling - relatively confident. Error messages are helpful for googling.
  • Asking your peers for help - If I know my peers are confident and have the capacity to provide assistance I will ask them directly.
  • Asking coaches for help - My coach has helped me out a few times and if I’m blocked I do ask him for help.
  • Improving your process with reflection - At the end of this sprint I will reflect on how I could have done things better. Especially with the way I manage my time.
JS Array Functions
.map()

Every value in the square brackets [] has a function applied to it, maybe a mathematic formula. Then the results are put in to new square brackets []. These brackets are called arrays.

.filter()

Same again. Every value in the square brackets [] has a function applied to it. Only this time the function has a test. If a value in the [] passes the test then it gets filtered and added to a new [].

.reduce()

This method reduces all the values in the array [] to one value. This is achieved by executing a provided function for each value of the array (from left-to-right). Rather then being added to an array the value is stored in an accumulator.
